# 虫豸阶段 MVP 开发计划

> **目标**：实现虫豸阶段的核心功能，让游戏能够"跑起来"，玩家可以体验到基本的生存压力和资源管理。

## MVP 核心功能清单

### 1. 回合系统
**目标**：建立游戏的基本时间节奏

#### 具体任务
- [ ] 实现回合计数器（当前回合数）
- [ ] 实现"下一回合"按钮
- [ ] 实现回合开始事件触发
- [ ] 实现回合结束结算逻辑
- [ ] 在UI上显示当前回合数

#### 技术要点
- 回合是抽象的时间单位，不对应现实时间
- 玩家手动点击"下一回合"按钮推进
- 回合内没有时间限制

#### 依赖系统
- Pinia状态管理（存储回合数）
- EventBus（触发回合事件）

---

### 2. 虫群生产与管理
**目标**：实现虫母生产虫群单位的完整流程

#### 具体任务
- [ ] 实现虫母生产Work任务（产卵）
- [ ] 实现卵对象（包含孵化时间）
- [ ] 实现卵孵化成幼虫的逻辑
- [ ] 实现幼虫对象（包含发育时间）
- [ ] 实现幼虫分化选择UI
- [ ] 实现幼虫发育成成虫的逻辑
- [ ] 实现工虫和兵虫的基础属性
- [ ] 实现虫群单位列表显示（BugGroupTile）

#### 技术要点
- 生产条件：环境好、食物充足
- 卵 → 幼虫 → 成虫的完整流程
- 玩家手动选择幼虫分化方向
- 特定环境 + 额外劳力可加快孵化

#### 依赖系统
- Work系统（已实现）
- Bug对象（已实现，需扩展）
- State系统（已实现）

---

### 3. 基础资源系统
**目标**：实现食物的生产与消耗机制

#### 具体任务
- [ ] 实现虫巢食物存储属性
- [ ] 实现虫群单位的食物消耗（每回合）
- [ ] 实现不同单位的消耗速度差异
- [ ] 实现虫母生产时的额外食物消耗
- [ ] 实现工虫劳动时的食物消耗
- [ ] 实现食物生产Work任务（觅食）
- [ ] 实现食物不足时的警告提示
- [ ] 在UI上显示当前食物存储量

#### 技术要点
- 虫群单位随时都会消耗食物
- 消耗速度不一致（如蜜罐虫消耗少，虫母生产时消耗多）
- 食物不足可能导致虫群单位死亡

#### 依赖系统
- State系统（已实现）
- Work系统（已实现）
- Bug对象（已实现）

---

### 4. 简单事件系统
**目标**：实现随机事件触发与天平机制

#### 具体任务
- [ ] 实现事件触发概率计算
- [ ] 实现天平机制（好事后坏事概率提高，反之亦然）
- [ ] 实现事件强度随机生成（-10到+10）
- [ ] 实现事件池（至少3-5个简单事件）
- [ ] 实现事件显示（EventTile）
- [ ] 实现事件效果执行
- [ ] 实现事件日志记录

#### 事件示例
- **好事**：发现食物（+食物）
- **坏事**：小地震（-虫巢耐久）
- **中性**：天气变化（影响觅食效率）

#### 技术要点
- 每回合开始时随机判断是否发生事件
- 发生事件后概率减少，不发生则概率增大
- 事件强度影响效果大小

#### 依赖系统
- Event对象（已实现基础架构）
- EventTile（已实现）
- 回合系统（MVP任务1）

---

### 5. 天敌威胁
**目标**：实现固定时间节点出现的敌人与简单战斗

#### 具体任务
- [ ] 实现敌人对象（复用Bug对象或创建新的Enemy对象）
- [ ] 实现天敌出现时间节点判断
- [ ] 实现天敌威胁递增机制
- [ ] 实现简单战斗系统（回合制，同时结算）
- [ ] 实现战斗UI（显示双方单位、选择行为）
- [ ] 实现战斗结果处理（伤亡、尸体）
- [ ] 实现逃跑机制（成功率由战力差距决定）
- [ ] 实现战斗日志记录

#### 战斗流程（简化版）
1. 天敌进攻虫巢
2. 进入战斗界面
3. 玩家选择虫群单位的行为（进攻/防御）
4. 所有单位同时结算
5. 判断战斗结束条件（一方全灭或逃跑成功）
6. 处理战斗结果

#### 技术要点
- 战斗回合和游戏回合是独立的
- 没有先后顺序，全部同时结算
- 大部分单位只能选择进攻，防御是技能
- 逃跑成功率由战力差距决定

#### 依赖系统
- Bug对象（已实现）
- State系统（已实现）
- 回合系统（MVP任务1）

## 开发顺序建议

### 第一阶段：基础框架
1. **回合系统**（任务1）
   - 最基础的系统，其他系统都依赖它

### 第二阶段：资源循环
2. **基础资源系统**（任务3）
   - 建立食物生产与消耗的基本循环
3. **虫群生产与管理**（任务2）
   - 实现虫群单位的生产流程

### 第三阶段：挑战与随机性
4. **简单事件系统**（任务4）
   - 增加游戏的随机性和变化
5. **天敌威胁**（任务5）
   - 增加游戏的挑战性和压力

## 验收标准

### MVP完成标准
玩家可以：
1. 点击"下一回合"按钮推进游戏
2. 虫母生产虫群单位（卵→幼虫→成虫）
3. 选择幼虫分化方向（工虫/兵虫）
4. 安排工虫觅食获取食物
5. 看到虫群单位消耗食物
6. 遇到随机事件并看到效果
7. 在固定回合后遇到天敌进攻
8. 进行简单战斗并看到结果

### 游戏体验目标
- 玩家能感受到基本的生存压力
- 玩家需要平衡食物生产与虫群扩张
- 玩家需要应对随机事件和天敌威胁
- 游戏有基本的策略深度
